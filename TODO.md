## 🚨 致命的なバグ（最優先課題: 前セッションからの引き継ぎ）

### 🐛 バーが一切動かせない（クリック・ドラッグ不能）問題
現在、画面上のタスクバー（アイテム）をクリック・ドラッグしようとしても、**Vis.jsがアイテムを認識せず、背後の空白・背景をクリックした判定（`item: null`, `what: null`）になってしまう** という致命的なバグが継続中です。

**【現在の状況と分かっていること】**
1. ログ出力APIで確認したところ、アイテム上をクリックしても `what: null`（背景判定）になる。
2. 考えられる原因として、CSSの `transform`、`calc` を用いたズーム機能（`--ui-scale`）や、TailwindCSSによる `box-sizing: border-box` リセットなどが、Vis.js 内部のアイテムサイズ（Bounding Box）計算を狂わせ、見た目の座標と実際の判定領域が完全に乖離している可能性が非常に高いです。
3. `xss: { disabled: true }` によるカスタムHTMLが複雑すぎることや、ダミーアイテム（`dummy-lane-item`）の干渉、さらには `index.html` に設定されていた `overflow-auto` などの要因も複合的に絡んでいる可能性があります。
4. 私（前任AI）がCSSから `pointer-events: none` を外したり、`.vis-item` の `z-index` を変更したり、Vis.jsのバージョンを `7.4.9` に固定したりと様々な試行錯誤を行いましたが、解決に至りませんでした。

## 🚀 次のステップ (NEXT TODO)

1. **アイテムのドラッグ不能バグの完全な原因究明と修正（最優先）**
   * CSSの影響（特に TailWind の影響や `--ui-scale` の影響）を切り分けるため、最小構成でのデバッグや、DOMインスペクタ等を用いた `vis-item` の実際のサイズ確認が必要です。
2. **タスクの上下レーンへのドラッグ＆ドロップ移動が正常に機能しない**
   * ドラッグが復旧した後に対応。空のレーンへドロップしようとすると弾かれる、あるいは意図したレーンに配置できない現象。
3. **親タスク（親ガントバー）の折りたたみ（Collapse）が機能しない**
   * 親バーをクリックしても子タスクの非表示・表示がトグルされない状態。
4. **タスク間の当たり判定（隙間へのテトリス配置）の不具合**
   * ドラッグ復旧後。見た目上は隙間が空いているにもかかわらず、Vis.jsが「重なっている」と誤判定し、下の行に押し出される問題。
5. **親バーの期間（開始日・終了日）が子タスクと1日ずれる問題**
   * 見た目上の終了日が1日分長く表示されてしまう（またはずれる）現象。
6. **マウスホイールによるUI拡縮・日付スケール変更の挙動不具合**
   * `Ctrl` + ホイールでのUI全体の拡大縮小、および単純ホイールでのタイムライン横軸（日付）のズーム挙動が不安定、あるいは機能しなくなる現象。
